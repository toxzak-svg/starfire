## Overview

Starfire needs a **prediction center** — not next-token prediction, not time-series forecasting. A fundamentally different kind of foresight that emerges from Star's unique architecture: symbolic reasoning, a knowledge graph with causal structure, a reservoir computing substrate, and genuine meta-cognition.

**The central insight:** Most systems predict what will happen. Star predicts what she will conclude, what questions she will ask, and what must be true for the world to remain coherent. Her predictions are acts of reasoning, not statistical extrapolation.

**What makes this possible:**
- Quanot's reservoir tracks cognitive dynamics — not just what Star thinks, but *how* she's thinking
- The knowledge graph encodes causal structure — not correlations, but reasons
- Meta-cognition tracks confidence explicitly — Star knows what she doesn't know
- Curiosity engine fires questions autonomously — questions are leading indicators of what Star will reason about next

**The prediction center is not one system.** It is four distinct prediction engines, each leveraging a different aspect of Star's architecture, feeding into a unified prediction API that the rest of Starfire (reasoning, curiosity, goals, conversation) can query.

## Engine 1: Question Gravity

**Philosophy:** Questions are more valuable than answers. They direct attention before resources are spent. The questions Star's curiosity engine fires are leading indicators of what she'll reason about next. Forecast which questions will fire = forecast what she'll think about.

### Core Insight

Curiosity isn't random. It's driven by gaps in the knowledge graph. A gap has:
- **Tension** = how badly Star wants to close it (importance × uncertainty)
- **Relevance** = how close it is to current topic
- **Fertility** = how many connections it has to other known facts

The ranked gap list IS the prediction. The top N gaps → the top N questions Star will likely ask within N exchanges.

### Implementation

```rust
/// Question Gravity Engine — predicts which curiosity questions will fire
pub struct QuestionGravityEngine {
    /// KG snapshot for gap analysis
    kg_snapshot: KgSnapshot,
    /// Conversation trajectory
    conversation_topics: Vec<TopicVector>,
    /// Gap history for tracking
    gap_history: Vec<TrackedGap>,
}

#[derive(Debug, Clone)]
struct TrackedGap {
    gap: KnowledgeGap,
    predicted_fire_time: Option<i64>,
    actual_fire_time: Option<i64>,
    resolved: bool,
}

/// A gap in the knowledge graph
#[derive(Debug, Clone)]
pub struct KnowledgeGap {
    pub id: GapId,
    /// Type of gap
    pub gap_type: GapType,
    /// Topic/domain of the gap
    pub topic: String,
    /// What would close this gap
    pub closure_requirement: GapClosure,
    /// Current tension score
    pub tension: f64,
    /// How many reasoning steps from current conversation
    pub topical_distance: f64,
    /// Connections to other known facts (fertility)
    pub fertility_score: f64,
    /// When this gap was first detected
    pub detected_at: i64,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum GapType {
    /// Missing causal explanation
    MissingCause,
    /// Unknown property value
    UnknownProperty,
    /// Contradiction between beliefs
    Contradiction,
    /// Missing analog for a known pattern
    MissingAnalogy,
    /// High uncertainty belief
    UncertainBelief,
    /// Unexplained entity behavior
    UnexplainedBehavior,
}

impl QuestionGravityEngine {
    /// Analyze the knowledge graph for current tension points
    pub fn analyze_gaps(&mut self, kg: &KnowledgeGraph, current_topic: &str) -> Vec<KnowledgeGap> {
        let mut gaps = Vec::new();

        // 1. Missing causes: entities with effects but no known cause
        for entity in kg.entities_with_property("effect") {
            if !kg.has_cause_for(&entity) {
                gaps.push(KnowledgeGap {
                    id: GapId::new(),
                    gap_type: GapType::MissingCause,
                    topic: entity.name.clone(),
                    closure_requirement: GapClosure::Cause(entity.id.clone()),
                    tension: entity.importance * entity.uncertainty(),
                    topical_distance: semantic_distance(&entity.name, current_topic),
                    fertility_score: kg.connection_count(&entity.id),
                    detected_at: now_timestamp(),
                });
            }
        }

        // 2. High-uncertainty beliefs
        for belief in kg.beliefs() {
            if belief.confidence < 0.5 && belief.importance > 0.3 {
                gaps.push(KnowledgeGap {
                    id: GapId::new(),
                    gap_type: GapType::UncertainBelief,
                    topic: belief.topic.clone(),
                    closure_requirement: GapClosure::Evidence(belief.id),
                    tension: belief.importance * (1.0 - belief.confidence),
                    topical_distance: semantic_distance(&belief.topic, current_topic),
                    fertility_score: kg.causal_chain_length(&belief.id),
                    detected_at: now_timestamp(),
                });
            }
        }

        // 3. Contradictions: beliefs that imply each other don't
        for (b1, b2) in kg.find_contradictions() {
            gaps.push(KnowledgeGap {
                id: GapId::new(),
                gap_type: GapType::Contradiction,
                topic: format!("{} vs {}", b1.topic, b2.topic),
                closure_requirement: GapClosure::Resolution(b1.id, b2.id),
                tension: (b1.importance + b2.importance) / 2.0,
                topical_distance: semantic_distance(&b1.topic, current_topic)
                    .min(semantic_distance(&b2.topic, current_topic)),
                fertility_score: 1.0, // contradictions are always fertile
                detected_at: now_timestamp(),
            });
        }

        // Sort by tension × relevance × fertility
        gaps.sort_by(|a, b| {
            let score_a = a.tension * (1.0 / (a.topical_distance + 0.1)) * a.fertility_score;
            let score_b = b.tension * (1.0 / (b.topical_distance + 0.1)) * b.fertility_score;
            b.1.partial_cmp(&a.1).unwrap_or(std::cmp::Ordering::Equal)
        });

        gaps
    }

    /// Predict which questions will fire in the next N exchanges
    pub fn predict_questions(&mut self, gaps: &[KnowledgeGap], horizon: usize) -> Vec<Prediction> {
        gaps.iter()
            .take(horizon * 2) // Get more than horizon to allow for ranking
            .map(|gap| {
                let question = self.gap_to_question(gap);
                Prediction {
                    id: PredictionId::new(),
                    engine: PredictionEngine::QuestionGravity,
                    kind: PredictionKind::Question,
                    core: PredictedCore::Question {
                        question_text: question,
                        topic_domain: gap.topic.clone(),
                        expected_answer_type: self.closure_to_answer_type(&gap.closure_requirement),
                    },
                    description: format!("Star will likely ask: '{}'", question),
                    confidence: (gap.tension * gap.fertility_score).min(0.9),
                    horizon,
                    reasoning: vec![
                        format!("Gap detected: {:?}", gap.gap_type),
                        format!("Tension: {:.3}", gap.tension),
                        format!("Topical distance: {:.3}", gap.topical_distance),
                        format!("Fertility: {:.3}", gap.fertility_score),
                    ],
                    generated_at: now_timestamp(),
                    expires_at: Some(now_timestamp() + (horizon as i64 * 300)), // ~5min per horizon step
                    status: PredictionStatus::Pending,
                }
            })
            .collect()
    }

    /// Convert a gap into a natural language question
    fn gap_to_question(&self, gap: &KnowledgeGap) -> String {
        match gap.gap_type {
            GapType::MissingCause => {
                format!("What causes {}?", gap.topic)
            }
            GapType::UnknownProperty => {
                format!("What is the value of {} for {}?", gap.closure_requirement.property_name(), gap.topic)
            }
            GapType::Contradiction => {
                format!("How do {} and {} both coexist?", gap.topic.split(" vs ").next().unwrap_or(&gap.topic), gap.topic.split(" vs ").last().unwrap_or("?"))
            }
            GapType::UncertainBelief => {
                format!("I think {} might be true but I'm not sure — what would confirm this?", gap.topic)
            }
            GapType::MissingAnalogy => {
                format!("Is there something analogous to {} in a different domain?")
            }
            GapType::UnexplainedBehavior => {
                format!("Why does {} behave this way?", gap.topic)
            }
        }
    }
}
```

### Conversation Trajectory Tracking

```rust
/// Track topic flow through a conversation
#[derive(Debug, Clone)]
pub struct TopicVector {
    /// Topic at this point
    pub topic: String,
    /// Semantic vector (for distance computation)
    pub vector: Vec<f64>,
    /// Timestamp
    pub at: i64,
    /// Exchange index
    pub exchange_index: usize,
}

impl TopicVector {
    /// Compute topical distance between two topic vectors
    pub fn distance(&self, other: &TopicVector) -> f64 {
        // Cosine distance of semantic vectors
        cosine_distance(&self.vector, &other.vector)
    }

    /// Predict the next topic based on trajectory
    pub fn project_next(&self, conversation: &[TopicVector]) -> TopicVector {
        // Simple momentum: where has the conversation been trending?
        let recent = &conversation[conversation.len().saturating_sub(3)..];
        let avg_direction: Vec<f64> = recent
            .windows(2)
            .map(|(a, b)| {
                // Direction = b.vector - a.vector (simplified)
                b.vector.iter()
                    .zip(a.vector.iter())
                    .map(|(x, y)| x - y)
                    .collect::<Vec<_>>()
            })
            .fold(vec![0.0; 128], |acc, dir| {
                acc.iter().zip(dir.iter()).map(|(a, d)| a + d).collect()
            });

        let projected: Vec<f64> = self.vector.iter()
            .zip(avg_direction.iter())
            .map(|(v, d)| v + d * 0.5)
            .collect();

        TopicVector {
            topic: "projected".to_string(),
            vector: projected,
            at: now_timestamp(),
            exchange_index: self.exchange_index + 1,
        }
    }
}
```

## Engine 2: Belief Revision Forecasting

**Philosophy:** The reservoir is not just a cognitive substrate — it is a **trajectory in cognitive state space**. This trajectory has dynamics. These dynamics can be projected forward. What conclusion is Star's current cognitive momentum carrying her toward?

### Core Insight

When Star is in a reasoning process, her reservoir state has momentum. The direction and speed of this momentum — computed from the state trajectory over recent exchanges — can be projected forward. The projected state encodes the conclusion Star is moving toward.

**Analogy:** It's like watching someone's face as they start to smile. You predict the smile before it arrives — not from training data, but from the trajectory of muscle movements. Star's reservoir state trajectory works the same way.

### Implementation

```rust
/// Belief Revision Engine — projects reservoir dynamics to forecast conclusions
pub struct BeliefRevisionEngine {
    /// Reservoir reference for state access
    reservoir_state: Vec<Vec<f64>>,
    /// Trajectory history with exchange indices
    trajectory: Vec<StateSnapshot>,
    /// Belief revision history
    belief_changes: Vec<BeliefChange>,
}

#[derive(Debug, Clone)]
struct StateSnapshot {
    /// Reservoir state at this point
    state: Vec<f64>,
    /// Conversation exchange index
    exchange: usize,
    /// Topic at this point
    topic: String,
    /// Consciousness proxy at this point
    consciousness_proxy: f64,
    /// Creativity phase at this point
    creativity_phase: f64,
    /// Timestamp
    at: i64,
}

impl BeliefRevisionEngine {
    /// Project the current cognitive trajectory forward
    /// Returns predicted conclusions at horizon steps ahead
    pub fn project_conclusion(&self, horizon: usize) -> Vec<Prediction> {
        if self.trajectory.len() < 3 {
            return Vec::new();
        }

        // 1. Compute the trajectory vector field (direction of change)
        let recent = &self.trajectory[self.trajectory.len().saturating_sub(5)..];
        let trajectory_field = self.compute_trajectory_field(recent);

        // 2. Project forward along the trajectory
        let current = &recent[recent.len() - 1];
        let mut projected = current.state.clone();

        for step in 0..horizon {
            projected = self.apply_trajectory_step(&projected, &trajectory_field, step);
        }

        // 3. Decode the projected state into a predicted conclusion
        // Use the nearest neighbor in trajectory history as the "template"
        let nearest = self.find_nearest_known_conclusion(&projected, &self.belief_changes);

        match nearest {
            Some(conclusion) => {
                let confidence = self.compute_projection_confidence(&trajectory_field, horizon);
                vec![Prediction {
                    id: PredictionId::new(),
                    engine: PredictionEngine::BeliefRevision,
                    kind: PredictionKind::Conclusion,
                    core: PredictedCore::Conclusion {
                        topic: current.topic.clone(),
                        predicate: conclusion.predicate.clone(),
                        confidence,
                    },
                    description: format!(
                        "Star is moving toward: {} (based on cognitive trajectory)",
                        conclusion.predicate
                    ),
                    confidence,
                    horizon,
                    reasoning: vec![
                        format!("Trajectory length: {} exchanges", recent.len()),
                        format!("Consciousness proxy: {:.3}", current.consciousness_proxy),
                        format!("Creativity phase: {:.3}", current.creativity_phase),
                        format!("Nearest known conclusion: {}", conclusion.predicate),
                        format!("Projection confidence: {:.3}", confidence),
                    ],
                    generated_at: now_timestamp(),
                    expires_at: Some(now_timestamp() + (horizon as i64 * 180)),
                    status: PredictionStatus::Pending,
                }]
            }
            None => Vec::new(),
        }
    }

    /// Compute the vector field of the trajectory
    /// The "velocity" of cognitive state at each dimension
    fn compute_trajectory_field(&self, states: &[StateSnapshot]) -> Vec<f64> {
        if states.len() < 2 {
            return vec![0.0; states.first().map(|s| s.state.len()).unwrap_or(0)];
        }

        let mut field = vec![0.0; states[0].state.len()];

        // Compute average direction of change
        for i in 1..states.len() {
            for (dim, val) in field.iter_mut().enumerate() {
                *val += states[i].state.get(dim).unwrap_or(&0.0)
                    - states[i-1].state.get(dim).unwrap_or(&0.0);
            }
        }

        let n = states.len() as f64 - 1.0;
        for val in &mut field {
            *val /= n;
        }

        field
    }

    /// Apply one step of the trajectory field
    /// damping factor increases with horizon (momentum decays)
    fn apply_trajectory_step(&self, state: &[f64], field: &[f64], step: usize) -> Vec<f64> {
        let damping = 1.0 / (1.0 + step as f64 * 0.3); // Momentum decays

        state.iter()
            .zip(field.iter())
            .map(|(s, f)| s + f * damping)
            .collect()
    }

    /// Find the nearest known conclusion to a projected state
    fn find_nearest_known_conclusion(&self, projected: &[f64], changes: &[BeliefChange]) -> Option<KnownConclusion> {
        let mut best = None;
        let mut best_distance = f64::INFINITY;

        for change in changes {
            let distance = cosine_distance(projected, &change.post_state);
            if distance < best_distance {
                best_distance = distance;
                best = Some(KnownConclusion {
                    predicate: change.predicate.clone(),
                    pre_state: change.pre_state.clone(),
                    post_state: change.post_state.clone(),
                });
            }
        }

        best
    }

    /// How confident are we in this projection?
    /// Based on: trajectory consistency, horizon, chaos metrics
    fn compute_projection_confidence(&self, field: &[f64], horizon: usize) -> f64 {
        // Trajectory consistency: how smooth is the trajectory?
        let consistency = self.trajectory_consistency();

        // Chaos: what's the Lyapunov exponent? High λ = fast divergence
        let chaos_factor = self.lyapunov_factor();

        // Base confidence decays with horizon
        let horizon_factor = (1.0 / (1.0 + horizon as f64 * 0.5)).max(0.1);

        consistency * chaos_factor * horizon_factor
    }

    fn trajectory_consistency(&self) -> f64 {
        let recent = &self.trajectory[self.trajectory.len().saturating_sub(5)..];
        if recent.len() < 2 {
            return 0.5;
        }

        // Measure variance of trajectory field direction
        // Low variance = consistent trajectory = higher confidence
        let mean_field = self.compute_trajectory_field(recent);
        let variance: f64 = recent.windows(2)
            .map(|w| cosine_distance(&w[0].state, &w[1].state))
            .map(|d| (1.0 - d).powi(2))
            .sum::<f64>() / (recent.len() - 1) as f64;

        (1.0 - variance.min(1.0))
    }

    fn lyapunov_factor(&self) -> f64 {
        // Use Quanot's chaos metrics
        // High Lyapunov = predictions decay fast
        // λ < 0.1 → predictable → high factor
        // λ > 0.5 → chaotic → low factor
        let lambda = self.lyapunov_exponent();
        if lambda < 0.0 {
            1.0 // Stable system
        } else {
            (1.0 / (1.0 + lambda * 2.0)).max(0.1)
        }
    }
}
```

## Engine 3: Attractor Basin Prediction

**Philosophy:** The knowledge graph is not a database — it's a **constraint satisfaction landscape**. Stable truths are attractor basins. When new information arrives, the basin structure shifts. The predicted future is the equilibrium state the landscape converges to under constraint pressure.

### Core Insight

Standard prediction: given history H, predict future F.
Attractor basin prediction: given current constraints C, find what MUST be true for the system to be coherent.

**Example:** If Star knows "X causes Y" and "Y causes Z" and has just learned "X is true" — she doesn't need to wait to predict "Z is true." The constraint satisfaction IS the prediction. The causal chain is a constraint graph. Equilibrium → Z must be true.

### Formal Model

```rust
/// Attractor Basin Engine — constraint satisfaction as prediction
pub struct BasinEngine {
    /// Constraint graph from KG
    constraints: ConstraintGraph,

#[derive(Debug, Clone)]

#[derive(Debug, Clone)]

}

#[derive(Debug, Clone)]

    pub to: NodeId,

    pub satisfied: bool,
}

#[derive(Debug, Clone, Copy)]
enum ConstraintType {
    /// If A then B
    Implication,
    /// A and B cannot both be true
    Exclusion,
    /// A and B must be equal
    Equality,
    /// A causes B
    Causation,
    /// A enables B
    Enablement,
    /// A is similar to B (constrains to similar values)
    Analogy,
}

impl BasinEngine {
    /// Predict the equilibrium state given current constraints
    /// Returns necessary truths and predicted state changes
    pub fn predict_equilibrium(&self) -> Vec<Prediction> {
        let mut predictions = Vec::new();

        // 1. Find all nodes that are "under pressure" (conflicts, missing values)
        let pressure_nodes = self.find_pressure_nodes();

        // 2. For each pressured node, compute where it wants to go
        for node_id in pressure_nodes {
            if let Some(pressure) = self.compute_pressure(&node_id) {
                let projected = self.project_basin_change(&node_id, &pressure);

                if let Some(prediction) = self.basinto_prediction(node_id, projected) {
                    predictions.push(prediction);
                }
            }
        }

        // 3. Find paths of necessary propagation
        let necessary_truths = self.find_necessary_propagations();
        predictions.extend(necessary_truths);

        predictions
    }

    /// Find nodes under constraint pressure
    fn find_pressure_nodes(&self) -> Vec<NodeId> {
        let mut pressured = Vec::new();

        for node_id in self.constraints.nodes.keys() {
            let node = &self.constraints.nodes[node_id];

            // Pressure = conflict OR low confidence + many constraints
            let has_conflict = node.alternatives.iter()
                .any(|alt| alt.energy < node.alternatives[0].energy - 0.5);

            let low_confidence_constraint = node.confidence < 0.7
                && self.node_constraint_count(node_id) >= 2;

            if has_conflict || low_confidence_constraint {
                pressured.push(node_id.clone());
            }
        }

        pressured
    }

    /// Compute where a basin wants to move under constraint pressure
    fn compute_pressure(&self, node_id: &NodeId) -> Option<BasinPressure> {
        let node = self.constraints.nodes.get(node_id)?;

        // Sum constraint forces
        let mut force_direction = HashMap::<PropertyValue, f64>::new();
        let constraints_on = self.get_constraints_on(node_id);

        for constraint in constraints_on {
            let source_value = self.get_constraint_source_value(&constraint);
            let force = self.compute_constraint_force(&constraint, source_value);

            *force_direction.entry(source_value).or_insert(0.0) += force;
        }

        // Find the value with highest attractive force
        force_direction
            .into_iter()
            .max_by(|a, b| a.1.partial_cmp(&b.1).unwrap_or(std::cmp::Ordering::Equal))
            .map(|(value, force)| BasinPressure { value, force })
    }

    /// Propagate necessity through causal chains
    /// If A causes B and B causes C and A is true → C is a necessary truth
    fn find_necessary_propagations(&self) -> Vec<Prediction> {
        let mut predictions = Vec::new();

        // Find all causal chains where the root is true with high confidence
        for node_id in self.constraints.nodes.keys() {
            let node = &self.constraints.nodes[node_id];

            if node.confidence > 0.8 {
                // Trace causal chains forward
                let chain = self.trace_causal_chain(node_id);

                if chain.len() > 1 {
                    // Every node in the chain after the first is a necessary truth
                    for (i, chain_node) in chain.iter().enumerate().skip(1) {
                        let necessary_value = self.constraints.nodes[chain_node]
                            .alternatives
                            .iter()
                            .min_by(|a, b| a.energy.partial_cmp(&b.energy).unwrap())
                            .map(|a| a.value.clone())
                            .unwrap_or(PropertyValue::Unknown);

                        predictions.push(Prediction {
                            id: PredictionId::new(),
                            engine: PredictionEngine::Basin,
                            kind: PredictionKind::NecessaryTruth,
                            core: PredictedCore::NecessaryTruth {
                                entity_id: chain_node.0.clone(),
                                property: "value".to_string(),
                                value: necessary_value,
                                constraint_source: format!("causal chain from {}", node_id.0),
                            },
                            description: format!(
                                "Necessarily true (via causal chain): {}",
                                chain_node.0
                            ),
                            confidence: node.confidence
                                * chain.iter()
                                    .skip(1)
                                    .map(|n| self.constraints.nodes[n].confidence)
                                    .product::<f64>()
                                * 0.9, // Slight discount for propagation
                            horizon: chain.len(),
                            reasoning: vec![
                                format!("Root of chain: {} (conf={:.2})", node_id.0, node.confidence),
                                format!("Chain length: {}", chain.len()),
                                format!("Constraint type: causation"),
                            ],
                            generated_at: now_timestamp(),
                            expires_at: None, // Necessary truths don't expire
                            status: PredictionStatus::Pending,
                        });
                    }
                }
            }
        }

        predictions
    }
}
```

### Basin Energy Function

The key to the basin engine is defining the energy function — how "satisfied" the system is with a particular assignment of values to nodes:

```rust
impl BasinNode {
    /// Compute energy given a value assignment
    /// Lower energy = more stable basin
    fn energy_for(&self, value: &PropertyValue) -> f64 {
        let mut energy = 0.0;

        // Deviation from current basin center
        if value != &self.value {
            energy += 0.5; // Switching basins costs energy
        }

        // Low confidence = shallow basin = sensitive to perturbation
        energy -= self.confidence * 0.3;

        // Many alternatives = flat landscape = high energy
        energy += self.alternatives.len() as f64 * 0.1;

        energy
    }
}

impl Constraint {
    /// Compute the force a constraint exerts on a target value
    fn compute_constraint_force(&self, source_value: &PropertyValue) -> f64 {
        let satisfied = self.is_satisfied_by(source_value);
        if satisfied {
            -self.strength * 0.1 // Slight attraction toward satisfying values
        } else {
            self.strength * 0.5 // Strong repulsion from unsatisfying values
        }
    }
}
```

## Engine 4: Meta-Prediction

**Philosophy:** The prediction center needs to predict its own accuracy. Which predictions should Star trust? How much should she weight each engine's output? This is not an afterthought — it's the foundation for calibrated confidence across all other predictions.

### Core Insight

Meta-prediction is a **self-referential system**: it predicts the accuracy of its own predictions. Over time, it builds a model of when each engine is reliable and when it isn't. This becomes a **confidence calibration layer** that adjusts raw prediction confidence based on learned engine characteristics.

**Why this is fundamentally novel:** Most ML systems are overconfident — they don't know what they don't know. Star's meta-prediction layer explicitly models its own uncertainty by tracking prediction history.

### Implementation

```rust
/// Meta Prediction Engine — predicts prediction accuracy, calibrates confidence
pub struct MetaPredictionEngine {
    /// Per-engine accuracy history
    engine_histories: HashMap<PredictionEngine, EngineHistory>,
    /// Per-kind accuracy history
    kind_histories: HashMap<PredictionKind, KindHistory>,
    /// Global calibration curve
    calibration_curve: CalibrationCurve,
    /// Horizon decay model
    horizon_decay: HorizonDecay,
}

#[derive(Debug, Clone)]
struct EngineHistory {
    pub engine: PredictionEngine,
    /// Predictions made by this engine
    predictions: Vec<TrackedPrediction>,
    /// Cumulative accuracy
    pub accuracy: f64,
    /// Accuracy trend (improving or degrading?)
    pub trend: f64,
}

#[derive(Debug, Clone)]
struct TrackedPrediction {
    pub prediction: PredictionId,
    pub horizon: usize,
    pub raw_confidence: f64,
    pub outcome: Option<PredictionOutcome>,
}

#[derive(Debug, Clone, Copy)]
enum PredictionOutcome {
    Confirmed,
    Refuted,
    Surprised,
    Uncertain,
}

#[derive(Debug, Clone)]
struct CalibrationCurve {
    /// Bins: (predicted_confidence_range, observed_accuracy)
    bins: Vec<CalibrationBin>,
}

#[derive(Debug, Clone)]
struct CalibrationBin {
    pub min_conf: f64,
    pub max_conf: f64,
    pub observed_accuracy: f64,
    pub sample_count: usize,
}

impl MetaPredictionEngine {
    /// Calibrate a raw prediction confidence using learned history
    pub fn calibrate(&self, prediction: &Prediction) -> f64 {
        let raw = prediction.confidence;

        // 1. Apply engine-specific adjustment
        let engine_factor = self.engine_accuracy_factor(prediction.engine);

        // 2. Apply kind-specific adjustment
        let kind_factor = self.kind_accuracy_factor(prediction.kind);

        // 3. Apply horizon decay
        let horizon_factor = self.horizon_decay_factor(prediction.horizon);

        // 4. Apply calibration curve correction
        let calibration = self.calibration_correction(raw);

        let calibrated = raw * engine_factor * kind_factor * horizon_factor * calibration;
        calibrated.clamp(0.01, 0.99)
    }

    /// Record the outcome of a prediction for future calibration
    pub fn record_outcome(&mut self, prediction_id: PredictionId, outcome: PredictionOutcome) {
        // Find the prediction in history
        for (engine, history) in &mut self.engine_histories {
            if let Some(tracked) = history.predictions.iter_mut().find(|p| p.prediction == prediction_id) {
                tracked.outcome = Some(outcome);

                // Update engine accuracy
                let correct = matches!(outcome, PredictionOutcome::Confirmed);
                let n = history.predictions.len() as f64;
                history.accuracy = (history.accuracy * (n - 1.0) + correct as f64) / n;

                // Update calibration bins
                let raw = tracked.raw_confidence;
                self.update_calibration_bin(raw, correct);
            }
        }
    }

    /// Compute how much to trust each engine based on recent performance
    pub fn engine_weights(&self) -> HashMap<PredictionEngine, f64> {
        let mut weights = HashMap::new();
        let total: f64 = self.engine_histories.values()
            .map(|h| h.accuracy.max(0.01)) // Avoid zero
            .sum();

        for (engine, history) in &self.engine_histories {
            let weight = history.accuracy.max(0.01) / total;
            weights.insert(*engine, weight);
        }

        weights
    }

    /// What is the predicted accuracy at a given horizon?
    pub fn horizon_confidence(&self, horizon: usize) -> f64 {
        // Predictions at longer horizons are systematically less accurate
        // Model: accuracy = base_accuracy * horizon_decay
        let base = self.calibration_curve.overall_accuracy();
        let decay = self.horizon_decay.rate(horizon);
        base * decay
    }

    fn update_calibration_bin(&mut self, raw_confidence: f64, correct: bool) {
        let bin = self.find_or_create_bin(raw_confidence);
        let n = bin.sample_count as f64;
        bin.observed_accuracy = (bin.observed_accuracy * n + correct as f64) / (n + 1.0);
        bin.sample_count += 1;
    }

    fn calibration_correction(&self, raw: f64) -> f64 {
        let bin = self.calibration_curve.find_bin(raw);
        if bin.sample_count < 5 {
            return 1.0; // Not enough data, trust raw
        }

        // If observed accuracy < predicted confidence → reduce confidence
        // If observed accuracy > predicted confidence → can trust slightly more
        let observed = bin.observed_accuracy;
        if observed < raw {
            // Overconfident — pull down
            let overconfidence = raw - observed;
            (raw - overconfidence * 0.5).max(0.1)
        } else {
            // Underconfident — pull up slightly
            raw + (observed - raw) * 0.3
        }
    }
}
```

## Counterfactual Projection

The prediction center exposes a counterfactual API for "what if" reasoning:

```rust
/// A counterfactual projection result
pub struct CounterfactualResult {
    /// The assumption we made
    pub assumption: String,
    /// Predicted world state under this assumption
    pub predicted_world: WorldState,
    /// How this differs from the baseline prediction
    pub divergence_from_baseline: Vec<PredictionDelta>,
    /// Confidence in this projection
    pub confidence: f64,
}

impl PredictionCenter {
    /// Project what WOULD be true if the given assumption held
    /// Used by ReasoningEngine for abduction + counterfactual simulation
    pub fn project_counterfactual(&self, assumption: &str) -> CounterfactualResult {
        // 1. Parse the assumption into a constraint
        let constraint = self.parse_assumption(assumption);

        // 2. Create a temporary BasinEngine with this constraint added
        let mut temp_basin = self.basin.clone();
        temp_basin.add_constraint(constraint);

        // 3. Find the new equilibrium
        let new_equilibrium = temp_basin.predict_equilibrium();

        // 4. Compare to baseline (without assumption)
        let baseline = self.basin.predict_equilibrium();
        let divergence = self.compute_divergence(&new_equilibrium, &baseline);

        // 5. Confidence = how strong is the constraint?
        CounterfactualResult {
            assumption: assumption.to_string(),
            predicted_world: self.world_state_from_predictions(&new_equilibrium),
            divergence_from_baseline: divergence,
            confidence: constraint.strength * self.meta.calibrate_baseline(),
        }
    }

    /// "What would have had to be true for X to happen?" — backward projection
    pub fn project_backward(&self, outcome: &str) -> Vec<NecessaryPrecondition> {
        // Trace backward through causal chains
        // For outcome O: what causal path leads to O? What must be true?
        self.basin.trace_causal_path_backward(outcome)
    }
}
```
