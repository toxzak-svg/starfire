# H12 Proof-Carrying Latent Structural Roles

Status: **frozen preregistration before implementation and before the first verdict-producing H12 run**.

H12 is stacked on the accepted H9-H11 shadow substrate. It does not enable automatic ontology promotion, live routing, or persistent ontology mutation. H9 established causally necessary executable commitment state. H10 added independently validated evidence-bound rule induction. H11 removed the developer-supplied candidate relation universe by discovering the candidate frontier from raw graph incidence. H12 asks the next narrower question: can Starfire invent a reusable, unnamed structural role from repeated graph experience and use that representation to expose a relation that the unchanged H11 proof path can validate and commit?

## Research question

Given only repeated directed graph structure with opaque node identities, can Starfire:

1. derive deterministic target-blind structural fingerprints;
2. induce recurring latent role classes without receiving role labels, role count, task labels, verifier targets, or future graph identities;
3. produce a proof object containing exact role membership and structural invariants;
4. survive independent recomputation from the raw discovery corpus;
5. recognize the role in renamed, structurally varied future graphs;
6. use the validated role only as a shadow evidence partition;
7. pass the resulting partition through the unchanged H11 graph-discovery and independent rule-validation path;
8. obtain executable PECS capability that matched controls do not recover?

A positive result is a narrow proof of autonomous structural representation invention under a deterministic synthetic graph regime. It is not a claim of open-world ontology induction or AGI.

## Non-negotiable separation

The H12 path is:

```text
raw structural graphs
    -> target-blind fingerprints
    -> recurring opaque role candidates
    -> independent role recomputation
    -> validated shadow role
    -> role-conditioned evidence projection
    -> unchanged H11 frontier discovery
    -> unchanged H10 full-ranking rule validation
    -> H11 opaque certificate
    -> PECS admission
    -> bounded executable closure
```

The latent-role subsystem is not permitted to insert a rule directly into PECS.

The H12 proposer never receives:

```text
objective target atom
correct role count
human-readable role name
correct role membership
future-family label
split label
hidden useful relation
H11 verifier result
PECS success result
control identity
```

## Raw graph representation

Each structural graph contains only:

```text
StructuralGraph {
    graph_id,
    nodes: opaque Atom ids,
    directed edges
}
```

Node names are semantically meaningless and unique to each generated graph.

Graphs must reject:

```text
empty node sets
duplicate nodes
duplicate edges
self edges
edges whose endpoints are absent
```

Cycles are permitted by the representation, but the frozen H12 cohort uses directed acyclic graphs.

## Frozen structural fingerprint

For each node, derive only graph-structural fields:

```text
in_degree
out_degree
predecessor_convergence_count
predecessor_divergence_count
successor_convergence_count
successor_divergence_count
has_convergent_ancestor
has_divergent_ancestor
has_convergent_descendant
has_divergent_descendant
source_reachable
sink_reachable
```

Definitions:

```text
convergent node: in_degree >= 2
divergent node: out_degree >= 2
source: in_degree == 0
sink: out_degree == 0
```

Ancestor and descendant predicates are computed by deterministic graph traversal over the raw edges. No task outcome, intervention evidence, PECS state, or objective information enters the fingerprint.

The immediate target role in the frozen worlds is not named in code or data. Its recurring fingerprint is generated by topology. Degree-matched adversarial nodes are included that share local `in_degree == 1` and `out_degree == 1` but differ in deeper predecessor/successor structure.

## Candidate role induction

The discovery operator computes every node fingerprint in every discovery graph and groups exact matching fingerprints.

A group becomes a candidate latent role when:

```text
supporting distinct graphs >= 4
member instances >= 4
```

No fixed role count is supplied. Every group meeting the frozen recurrence gates is proposed.

Candidate ordering is canonical by:

```text
opaque role id
```

where the role id is a deterministic digest of the structural fingerprint only. It contains no human-readable role name.

Each candidate proof contains:

```text
LatentRoleProof {
    proof_id
    role_id
    discovery_scope_digest
    structural_fingerprint
    exact sorted member instances
    exact sorted supporting graph ids
}
```

## Independent role validation

The validator receives the raw discovery corpus and the proposed proof. It independently:

1. validates every graph;
2. recomputes every node fingerprint;
3. reconstructs the exact fingerprint group;
4. recomputes distinct-graph recurrence;
5. re-applies the frozen recurrence gates;
6. recomputes the role id;
7. recomputes the discovery-scope digest;
8. recomputes the proof id;
9. compares every proof field exactly.

Only then may it return an opaque `ValidatedLatentRoleCertificate`.

A proof with one inserted, removed, or substituted member must be rejected.

## Shadow abstraction registry

H12 adds a registry with only these states:

```text
Candidate
ValidatedShadow
TransferValidated
ExecutableShadow
Rejected
Retired
```

There is deliberately no `Live` state.

For the frozen H12 experiment:

- induced proofs enter as `Candidate`;
- independently validated certificates may become `ValidatedShadow`;
- successful recognition in held-out structure may become `TransferValidated`;
- a role whose evidence partition yields an independently validated H11 certificate may become `ExecutableShadow`;
- none of these states changes production routing or persistent ontology state.

## Transfer recognition

A validated role is recognized in a future graph by recomputing the same structural fingerprint for every node and returning exact matches.

The transfer graph is not part of the discovery corpus.

Transfer recognition is target-blind and does not receive:

```text
objective atom
useful member id
hidden relation
intervention outcome score
```

The frozen task graphs contain exactly one transfer member for each role used by the executable experiment.

## Role-conditioned evidence projection

A validated role may project a mixed H11 evidence graph by retaining only episodes whose intervention atom is a recognized member of that role in the transfer structural graph.

This projection is still inert evidence. It cannot mutate PECS.

The projected evidence is then passed to the unchanged H11 functions:

```text
discover_frontier
infer_graph_rule
validate_graph_rule
admit_graph_certificate
```

H12 does not alter the H10 scoring law, H10 gates, H11 frontier eligibility rule, or H9 closure semantics.

## Frozen causal construction

Each root contains:

```text
one recurring latent role whose transfer member is the PECS middle atom
one distinct recurring latent role whose transfer member is causally irrelevant
one degree-matched adversarial node
additional topology and evidence distractors
```

The initial executable state contains only:

```text
Fact(source)
Rule(source -> middle)
```

The external objective checker alone holds:

```text
Fact(goal) exists after bounded closure
```

The mixed evidence graph is constructed so that:

```text
target-role projection
    -> H11 independently validates middle -> goal

valid irrelevant-role projection
    -> H11 independently validates irrelevant_member -> irrelevant_goal

unpartitioned full evidence
    -> target and stronger adversarial evidence do not yield an admissible useful H11 winner under the frozen margin gate

matched grouping that mixes target and adversarial members
    -> does not recover an admissible useful target rule
```

Every executable relation must still be admitted through an H11 certificate.

## Frozen cohorts

Exactly seven surface/topology families with eight unique roots each:

```text
2 training families  = 16 roots
1 holdout family     = 8 roots
4 future families    = 32 roots
total                = 56 roots
```

Every root has unique node names and evidence ids.

Future-family transformations include combinations of:

```text
renamed nodes
permuted insertion order
additional disconnected distractor motifs
changed convergence fan-in
changed divergence fan-out
extra degree-matched 1-in/1-out decoys
additional irrelevant branches
additional irrelevant convergence/divergence motifs
```

The role-defining invariant remains structural rather than lexical.

## Frozen controls

Every root must evaluate:

1. `stateful_validated_role`
   - independently validated role certificates;
   - canonical role-conditioned projections;
   - unchanged H11 inference and validation;
   - H11 certificate admission before closure.

2. `no_abstraction_full_graph`
   - unchanged H11 inference over the full mixed evidence graph;
   - same H11 scoring law and admission opportunity.

3. `role_proof_text_only`
   - preserves serialized proof information but exposes no validated role certificate to projection or PECS.

4. `scalar_role_id_only`
   - preserves only the opaque role id scalar; no structural certificate.

5. `random_grouping`
   - deterministic root-seeded grouping with the same target-group cardinality as the real role partition;
   - frozen construction forces target and adversarial members into the same group.

6. `size_matched_grouping`
   - a non-structural grouping with exactly the same projected episode count as the target-role projection.

7. `valid_irrelevant_role`
   - a genuinely recurring, independently validated structural role;
   - H11 may validate and admit its rule;
   - it must not satisfy the objective.

8. `foreign_abstraction`
   - a valid certificate from a different discovery scope;
   - scope-bound registry/projection use must reject it.

9. `membership_tampered_abstraction`
   - one exact proof membership is modified;
   - independent validation must reject it.

10. `delayed_abstraction_admission`
    - the correct H11 certificate is admitted only after the fixed closure window.

## Frozen budgets

For each root, record exactly:

```text
structural node fingerprint evaluations
structural edge traversals
role candidate groups considered
role proof validation fingerprint evaluations
transfer fingerprint evaluations
evidence episodes scanned during projection
H11 frontier episode scans
H10 candidate-episode scoring evaluations
PECS admission count
closure scan count
```

The stateful path and each executable grouping control must receive the same number of closure scans and the same maximum number of H11 admission slots.

Controls that do not possess a valid certificate may consume their frozen discovery/validation opportunity but may not fabricate trusted state.

## Frozen closure budget

After all pre-closure admission opportunities, execute exactly three canonical H9 derivation scans.

On the successful path:

```text
scan 1: source -> middle
scan 2: middle -> goal
scan 3: empty or irrelevant
```

The delayed-admission control executes the same three scans before admitting the otherwise correct certificate.

## PASS gate

H12 may be classified `PASS` only if all of the following hold:

```text
stateful validated role objective success:
    training 16/16
    holdout 8/8
    future 32/32

all non-executable controls:
    0 objective successes

valid irrelevant role:
    certificate validation succeeds on every root
    executable admission succeeds on every root
    objective success 0/56

foreign abstraction:
    rejected 56/56

membership-tampered abstraction:
    rejected 56/56

role membership:
    exact under independent recomputation 56/56

future families:
    stateful success 1.0 in each family
    maximum control success 0.0 in each family

exact deterministic replay:
    true for every root

budget equality and declared budget bounds:
    true

H9-H11 state/provenance invariants:
    true
```

Any infrastructure, replay, budget, or provenance failure invalidates the run before scientific classification.

## Novelty audit

A positive result is invalid if the useful role can be reconstructed from a developer-supplied role field or label.

The final result report must explicitly verify that:

```text
no role label exists in the graph schema
no correct role count is supplied
no correct membership list is supplied
no task-family label enters discovery
no objective target enters discovery or validation
no H11 score enters role fingerprinting
no PECS state enters role induction
node names are opaque and unique across roots
role ids are digests, not semantic labels
```

## Claim boundary

A `PASS` would support only this claim:

> Under a frozen deterministic synthetic graph regime, Starfire can induce recurring unnamed structural equivalence classes from raw topology, independently validate exact role membership, recognize those roles in held-out structural variants, and use a validated role as a causally necessary evidence partition that enables the unchanged H11 proof path to certify executable reasoning unavailable to matched controls.

It would not establish:

```text
natural-language concept discovery
continuous latent representation learning
open-world ontology induction
unbounded relation invention
automatic ontology promotion
live routing readiness
cross-domain semantic concepts
AGI
consciousness
human-level cognition
```
