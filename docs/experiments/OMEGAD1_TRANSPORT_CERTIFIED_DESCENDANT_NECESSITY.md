# ΩD1 — Transport-Certified Descendant Necessity

Status: preregistered before implementation and before any verdict-producing run.

## Lineage and motivation

Ω1 showed that an independently validated executable refinement can repair an order-blind state alias on a fixed discovery support.

Ω2 then created a genuine second-generation descendant language and passed every necessity/control gate, but the cumulative chain failed withheld transfer because Ω1 had selected a discovery-equivalent but transformation-unstable ancestor representative.

ΩR1 separately established a new foundation result: among 72 executable programs that were identical on the discovery behavioral partition, same-history evaluation over a preregistered structure-preserving transformation orbit selected a zero-violation within-pair representative that transferred perfectly to unseen block permutations. Partition-only and equal-compute target-stationary controls failed exactly as predicted.

ΩD1 asks the next narrow question:

> Does a transport-certified executable ancestor create a descendant hypothesis language that is genuinely necessary for a second proof-carrying refinement, and does the resulting two-generation chain transfer to structural transformations not used during ancestor transport calibration?

ΩD1 is a new experiment. It is not a rerun or rescue of Ω2.

## Primary causal chain

```text
L0
  -> stage-1 alias defect
  -> synthesize transport-aware ancestor Δ1
  -> independently validate transport certificate C1
  -> admit C1 into executable ancestor state
  -> L1 = L0 + transport-certified Δ1
  -> descendant hypothesis language becomes non-empty
  -> synthesize descendant Δ2
  -> independently validate Δ2 against exact L1 identity
  -> admit Δ2 into layered executable state
  -> L2
  -> predict on structural transformations unseen by both ancestor calibration and descendant fitting
```

The central necessity claim requires all of the following:

```text
H(L0) contains zero descendant programs
L0 raw grammar cannot express the stage-2 behavioral partition
correct transport-certified Δ1 makes Δ2 expressible
wrong but valid transport-certified ancestor does not
replacing exact Δ1 before Δ2 validation invalidates the descendant proof under matched full compute
transport-certificate payload without admission does not create a descendant terminal
descendant payload without admission does not create downstream capability
stationary-calibrated accidental ancestor can support discovery-time Δ2 validation but fails cumulative held-out transfer
```

## Raw fixture

Each root has eight raw atoms arranged as four ordered pairs:

```text
(A, B) -> latent pair-order bit p
(C, D) -> q
(E, F) -> r
(G, H) -> s
```

Discovery exhausts the complete Boolean cube:

```text
(p, q, r, s) ∈ {0,1}^4
```

for exactly 16 histories per root.

Discovery block order is fixed:

```text
AB | CD | EF | GH
```

Every discovery history therefore has the same eight raw atoms, the same atom counts, and the same order-blind base multiset key.

The hidden bits are used only to generate witnessed fixture outcomes. Synthesis, transport scoring, descendant enumeration, validation, executable state, and downstream prediction never receive `p/q/r/s` labels.

## Frozen descendant grammar

The descendant language is generated from current executable ancestor state:

```text
ancestor_bit := output of one actually admitted transport-certified ancestor refinement
raw_bit      := output of one Ω1 raw program

descendant_program :=
      ancestor_bit == raw_bit
    | ancestor_bit != raw_bit
```

The grammar itself is developer-supplied and frozen. The ancestor terminal is not developer-supplied: it exists only when a transport certificate has been independently validated and admitted to the current root-bound ancestor state.

Therefore:

```text
H(L0): 0 admitted ancestor bits -> 0 descendant programs
H(L1): 1 admitted ancestor bit  -> 2 × 828 = 1,656 descendant programs
```

No proof payload, endpoint-computed bit, rejected certificate, foreign certificate, or text label may create an ancestor terminal.

## Descendant search

For one admitted ancestor:

1. derive the eight-atom raw vocabulary from stage-2 discovery histories;
2. enumerate all 828 raw Ω1 programs;
3. pair the admitted ancestor bit with every raw program under `==` and `!=`;
4. execute all 1,656 descendant candidates over all 16 discovery histories;
5. canonicalize binary partitions up to Boolean complement;
6. rank partitions by separation of all 64 independently witnessed opposite-outcome pairs;
7. break ties by minimum partition support, then canonical program string.

Frozen correct-ancestor result:

```text
ancestor terminals = 1

descendant program/history executions = 26,496
unique descendant partitions = 5
winner repaired pairs = 64/64
runner-up repaired pairs = 32/64
winner margin = 32
partition support min = 8
```

The target partition is expressible up to Boolean complement by combining the transport-certified `p` ancestor with a raw predicate for the `r` pair order.

Frozen wrong-ancestor ceiling:

```text

full 1,656-candidate search

```

## Required interpretation of PASS

A PASS would support only:

> Under the frozen symbolic pair-block regime, an independently validated and admitted transport-stable ancestor refinement created an executable hypothesis-language terminal that did not exist in L0; the new two-bit descendant partition was not expressible by any single raw Ω1 program, but became expressible and independently valid under the exact transport-certified ancestor state. A separately transport-certified wrong ancestor and exact ancestor replacement both failed under matched full descendant-search compute, while a discovery-valid but stationary-calibrated accidental ancestor supported descendant fitting yet failed cumulative held-out transfer. Timely descendant admission was necessary for the final capability.

## Claims explicitly not established

A PASS would not establish:

- automatic discovery of transformation groups;
- learned invention of the descendant grammar itself;
- arbitrary-depth recursive representation growth;
- grammar mutation;
- natural-language concept formation;
- open-world semantic equivalence;
- safe production self-modification;
- AGI;
- consciousness;
- human-level cognition.

## Next step after PASS

A PASS would justify a deeper experiment in which a third-generation refinement requires a composition operator not present in the frozen descendant grammar, so the system must propose and independently validate a grammar extension rather than merely instantiate another program from a developer-fixed language.
